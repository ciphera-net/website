# Contributing to Ciphera Website

Thank you for your interest in contributing to the Ciphera website! This document provides guidelines for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Respect different viewpoints

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Create a branch for your changes
5. Make your changes
6. Test your changes
7. Submit a pull request

## Development Setup

See the main [README.md](../README.md) for setup instructions.

## Coding Standards

### TypeScript

- Use TypeScript for all new code
- Follow the existing code style
- Use meaningful variable names
- Add comments for complex logic

### React

- Use functional components with hooks
- Keep components focused and small
- Use TypeScript for props

### Styling

- Use Tailwind CSS for styling
- Follow the brand guidelines (Orange #FD5E0F as accent only)
- Ensure dark mode support on all components
- Maintain design system consistency

## Pull Request Process

1. Update documentation if needed
2. Ensure all code follows TypeScript best practices
3. Test in multiple browsers
4. Maintain design system consistency
5. Submit PR with clear description

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add new product showcase section
fix: Resolve mobile menu styling issue
docs: Update README with deployment instructions
style: Improve hero section animations
```

## Testing

- Test in multiple browsers (Chrome, Firefox, Safari, Edge)
- Test on mobile devices
- Verify dark mode functionality
- Check responsive design at different breakpoints

## Design System

When making changes, ensure:

- Brand orange (#FD5E0F) is used as accent only
- Neutral color scale (50-900) is used for UI elements
- Plus Jakarta Sans font is used throughout
- Rounded corners match existing patterns (xl, 3xl)
- Smooth transitions (200ms, 300ms) are maintained
- Dark mode support is included

## Questions?

Feel free to open an issue for questions or discussions.
